Maven

According to Maven 3 Tutorial – Project Setup:
Maven is a software project management and comprehension tool that includes: build tools, dependency management, project reporting and much more.  I say “much more” because at the core Maven is a plugin execution framework.  There are plugins supported by the Maven project, plugins supported by Mojo Project, and third party plugins.
About the comparisons between Ant and Ivy, the same tutorial linked above says that they all are different tools:
Ant is a build tool and Ivy is a dependency management tool.  Maven is not just a build tool or just a dependency management tool.  On the contrary, Maven is a project management tool that embodies software development best practices.

Maven Plugins

Crucial or important maven plugins include:
  • maven-compiler-plugin, "used to compile the sources of your project".
  • maven-war-plugin, "responsible for collecting all artifact dependencies, classes and resources of the web application and packaging them into a web application archive".
  • maven-jetty-plugin.
These may also be proved important:
  • maven-release-plugin, "used to release a project with Maven, saving a lot of repetitive, manual work".
  • maven-assembly-plugin "allow[s] users to aggregate the project output along with its dependencies, modules, site documentation, and other files into a single distributable archive".

Jetty

In order to use Jetty with Maven, maven-war-plugin and maven-jetty-plugin are prerequisites.

Path Variables Configuration in Mac OS X Lion

This is done with the export command, as: [sourcecode language="bash"] export M2_HOME=/usr/share/maven [/sourcecode] It will load the newly configured M2_HOME in the memory. In order to make it persistent, save it in a bash_profile file (in your home directory) and reload the profile making the change persistence. First update the path variable: [sourcecode language="bash"] echo "export M2_HOME=/usr/share/maven" >> .bash_profile [/sourcecode] Then make it persistent: [sourcecode language="bash"] . .bash_profile [/sourcecode]

Basic Project Resources

The pom.xml

Usually a pom.xml begins with the project definition [sourcecode language="xml"] 4.0.0 com.github.rafaelfiume SalumeSupplier 0.0.1-SNAPSHOT war Salume Supplier Server-side application to help sell mortadelas [/sourcecode]

Dependencies

Dependencies, like JUnit and Commons Lang libraries, are defined in a project through the dependency element in a pom.xml file: [sourcecode language="xml"] commons-lang commons-lang 20030203.000129 junit junit 4.10 test [/sourcecode] Follows all the necessaries dependencies to start a project that uses Spring 3 artifacts with Maven: [sourcecode language="xml"] org.springframework spring-context ${org.springframework.version} org.springframework spring-web ${org.springframework.version} org.springframework spring-webmvc ${org.springframework.version} org.springframework spring-test ${org.springframework.version} test [/sourcecode] The ${org.springframework.version} variable is definided as a property element. Another property defined is the source encoding: [sourcecode language="xml"] UTF-8 3.1.1.RELEASE [/sourcecode]
